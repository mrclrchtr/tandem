After supi_flow_start creates a ticket, the agent (and user) see only the ticket ID and status. But knowing the actual filesystem path (`.tndm/tickets/TNDM-XXXXXX/`) is useful for manual inspection, debugging, and file-level operations. We should look up the ticket via `tndm ticket show --json` after creation, derive the directory from `content_path`, and include it in the response text and details.
